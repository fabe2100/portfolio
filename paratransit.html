Accessible Transit Booking Project

A university project focused on designing and developing a website for online booking of paratransit services (färdtjänst) for a municipality.

Project Overview

This project involved the development of a website to enable residents to book paratransit trips online. The primary goals were to create a user-friendly and accessible platform, adhering to WCAG AA guidelines, and to apply a user-centered, iterative design process.

The challenge was to understand and address the needs of users with diverse abilities and requirements. This was achieved through user research, persona creation, and scenario development.

  • User login and account management [cite: 31]

Design Process

The design process followed a user-centered approach, incorporating the following stages:

  • **Research:** Gathering information on paratransit services and user needs.
  • **User Analysis:** Developing user personas and scenarios to understand user behavior and requirements.
  • **Prototyping:** Creating interactive prototypes in Figma to test and refine the user interface. [cite: 270, 271]
  • **Development:** Implementing the design using HTML, CSS, and JavaScript, with a focus on accessibility and responsiveness. [cite: 49]
  • **Testing:** Conducting user testing to evaluate usability and identify areas for improvement. [cite: 223]

Challenges and Solutions

A key challenge was ensuring the website was accessible to users with various disabilities. Solutions included:

  • Implementing sufficient color contrast. [cite: 240, 241, 242, 243, 244, 245]
  • Providing clear and concise text. [cite: 240, 241, 242, 243, 244, 245]
  • Ensuring keyboard navigation using tabindex. [cite: 245, 246, 247, 43]
  • Using ARIA attributes for screen reader compatibility. [cite: 245, 246, 247, 248, 44, 45]

Another challenge was to create an intuitive booking process. This was addressed through iterative prototyping and user testing, which helped to simplify the forms and navigation. [cite: 223, 57, 58, 59]
